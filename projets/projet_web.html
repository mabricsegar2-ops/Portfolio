Présentation

TP noté de Programmation Web (R4.09), réalisé seul, autour du cas « La Fnuc » : une librairie en ligne (consulter un livre, le commander, gérer le stock et l'historique d'un client). L'enjeu : implémenter le même besoin de trois façons pour comparer les approches, et rendre chaque version connectable à plusieurs SGBD.

  • CGI Python (Common Gateway Interface) accédant directement à MySQL ;
  • CGI avec SQLAlchemy, connectable à MySQL, PostgreSQL, Oracle et SQLite ;
  • Application Flask exposant une API REST consommée côté client en JavaScript/AJAX.

CGI Python + MySQL

  • myshowbook · affiche un livre (titre, auteurs, prix, couverture, résumé) et propose un lien vers la commande en transmettant le bookId.
  • myorderbook · formulaire de commande (login, mot de passe, quantité) : vérification du stock, création du client si absent, insertion de la commande, mise à jour du stock, avec gestion du commit / rollback.
  • Correction du schéma MySQL · ajout d'AUTO_INCREMENT sur les clés, nettoyage des entrées invalides (ID = 0, commandes orphelines) et reconstruction de la contrainte de clé étrangère.

CGI avec SQLAlchemy (multi-bases)

  • Mêmes fonctions (affichage + commande), mais via SQLAlchemy : un seul code pour MySQL, PostgreSQL, Oracle et SQLite.
  • DSN construit dynamiquement selon le paramètre CGI db=, qui est ensuite propagé dans toute la chaîne d'appels pour conserver le même moteur.
  • Transactions gérées par engine.begin() (commit / rollback automatique) et mise à jour du chiffre d'affaires cumulé du client.

Application Flask · API REST + AJAX

Version « web légère » : une API REST (approche Code First) sert les données, et une couche JavaScript/AJAX construit les pages côté client en manipulant le DOM.

  • Points de terminaison complets : /books, /customers, /orders, /inventories et leurs relations (/books/<id>/topics, /keywords, /orders…), avec linking et embedding.
  • Implémentés avec SQLAlchemy (SQL Expression Language) et reflection des tables existantes ; verbes GET / POST / PATCH / DELETE.
  • Côté client : commande d'un livre (vérification stock → POST /orders → PATCH /inventories), historique des commandes et calcul HT / TVA / TTC.

Tests & documentation de l'API

L'API a été documentée et rendue testable depuis le navigateur via Swagger UI (route /docs), et vérifiée manuellement avec Postman sur l'ensemble des routes (collections de requêtes GET / POST / PATCH / DELETE).

CGI MySQL · commande transactionnelle (Python)

Requêtes paramétrées (anti-injection), création du client, commande, mise à jour du stock, commit / rollback.

-- Nettoyage des donnees invalides avant migration
DELETE FROM commandes WHERE CLIENT NOT IN (SELECT ID FROM clients);
DELETE FROM commandes WHERE ID = 0;
DELETE FROM clients   WHERE ID = 0;

-- On retire la cle etrangere le temps de la migration
ALTER TABLE commandes DROP FOREIGN KEY commandes_ibfk_2;

-- Activation de l'AUTO_INCREMENT sur les cles primaires
ALTER TABLE clients   MODIFY ID INT AUTO_INCREMENT;
ALTER TABLE commandes MODIFY ID INT AUTO_INCREMENT;

-- Reconstruction de la contrainte de cle etrangere
ALTER TABLE commandes ADD CONSTRAINT commandes_ibfk_2
    FOREIGN KEY (CLIENT) REFERENCES clients(ID);

# API REST : acces generique via SQLAlchemy (SEL) + reflection des tables
metadata.reflect(bind=engine)

def get_one(table_name, column_name, value):
    table = metadata.tables[table_name]
    rows = connection.execute(
        sqlalchemy.select([table]).where(
            getattr(table.c, column_name) == value)).fetchall()
    return {"result": [dict(r) for r in rows]}

API REST (Flask + SQLAlchemy)

Accès générique aux tables réfléchies, exposé en GET / POST / PATCH / DELETE.

async function processOrder(form) {
    event.preventDefault();
    const article  = form.article.value;
    const quantite = parseInt(form.quantite.value);

    // 1. Verification du stock (livre + inventaire embarque)
    const dataBook  = await (await fetch("/books/" + article)).json();
    const inventory = dataBook.result[0].inventory;
    if (inventory.niveau < quantite) {
        alert("Stock insuffisant. Reste : " + inventory.niveau);
        return false;
    }
    // 2. Creation de la commande
    await fetch("/orders", {
        method: "POST",
        headers: { 'Content-type': "application/json" },
        body: JSON.stringify({ article, client: form.client.value,
                               datecom: form.datecom.value, quantite })
    });
    // 3. Baisse du stock
    await fetch("/inventories/" + inventory.id, {
        method: "PATCH",
        headers: { 'Content-type': "application/json" },
        body: JSON.stringify({ niveau: inventory.niveau - quantite })
    });
    alert("Commande validee avec succes !");
}
