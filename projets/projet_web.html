Présentation du projet

Développement d'une application web de gestion d'une librairie en ligne (consultation de livres, commande, gestion du stock et historique client). Le projet couvre toute la chaîne, de la base de données relationnelle à l'interface, en passant par une API REST.

Il m'a permis de mettre en pratique plusieurs approches d'accès aux données et de comparer leurs intérêts respectifs.

Travail réalisé

  • CGI Python + MySQL : affichage et commande de livres, vérification du stock, création de client, gestion du commit/rollback
  • SQLAlchemy : accès multi-bases (MySQL, PostgreSQL, Oracle, SQLite) via un DSN construit dynamiquement
  • API REST avec Flask : points de terminaison /books, /customers, /orders, /inventories et leurs relations
  • AJAX (JavaScript) : commande et mise à jour du stock côté client, calcul HT/TVA/TTC, affichage de l'historique des commandes
  • Adaptation du schéma MySQL (clés auto-incrémentées, contraintes de clés étrangères, nettoyage des données invalides)

Apprentissages critiques acquis

AC14.04 | Comprendre le cycle de vie d'un projet informatique

De la conception de la base à l'interface en passant par l'API, j'ai suivi les étapes d'un projet de développement complet et structuré.

AC21.05 | Apprécier l'intérêt de briques logicielles existantes et savoir les utiliser

L'utilisation de Flask et de l'ORM SQLAlchemy m'a montré l'intérêt de s'appuyer sur des briques logicielles éprouvées plutôt que de tout réécrire, notamment pour gérer plusieurs SGBD avec un même code.

AC21.04 | Comprendre la nécessité de tester, corriger et documenter un programme

La gestion du commit/rollback, le nettoyage des données invalides et la documentation des rendus m'ont appris l'importance de la robustesse et de la traçabilité d'un programme.

Réflexion personnelle

Ce projet m'a fait découvrir la logique des API REST et l'intérêt de séparer la donnée (back-end) de son affichage (front-end via AJAX).

Pouvoir cibler plusieurs SGBD avec un même code grâce à un ORM est une compétence directement réutilisable pour développer des composants de solutions décisionnelles.
